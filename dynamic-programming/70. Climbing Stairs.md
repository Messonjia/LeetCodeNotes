# LC 70 · Climbing Stairs

## Problem Statement

You are climbing a staircase. It takes `n` steps to reach the top.  
Each time you can either climb `1` or `2` steps.  
In how many **distinct ways** can you climb to the top?

**Constraints:** `1 <= n <= 45`

---

## Key Insight — This Is Fibonacci in Disguise

At any step `i`, you could have arrived from:
- Step `i-1` (took 1 step), or
- Step `i-2` (took 2 steps)

So the number of ways to reach step `i` is:

```
ways(i) = ways(i-1) + ways(i-2)
```

This is exactly the Fibonacci recurrence. Climb Stairs is **Fibonacci shifted by one**:

| n       | 1 | 2 | 3 | 4 | 5 | 6  |
|---------|---|---|---|---|---|----|
| ways(n) | 1 | 2 | 3 | 5 | 8 | 13 |
| F(n+1)  | 1 | 2 | 3 | 5 | 8 | 13 |

`ways(n) = F(n+1)` — the answer is always the next Fibonacci number.

---

## Solution — Space-Optimized DP (Size-2 Window)

```python
class Solution:
    def climbStairs(self, n: int) -> int:
        if n == 1:
            return 1

        dp = [1, 2]               # dp[0] = ways(i-1), dp[1] = ways(i)

        for i in range(3, n + 1): # base cases ways(1) and ways(2) are pre-seeded
            total = dp[0] + dp[1]
            dp[0] = dp[1]
            dp[1] = total

        return dp[1]
```

---

## ⚠️ Common Mistake — Unnecessary `dp` Size

The original approach allocates `dp = [0] * 3` (size 3), but `dp[0]` is **never read or written** after initialization — it's always `0` and serves no purpose.

```python
# ❌ Wasteful — dp[0] is dead weight
dp = [0] * 3
dp[1] = 1
dp[2] = 2
...
return dp[2]

# ✅ Clean — matches the LC 509 rolling window pattern exactly
dp = [1, 2]
...
return dp[1]
```

Reducing to size 2 makes the intent clearer and directly mirrors the LC 509 Fibonacci solution.

---

## Comparison — LC 70 vs LC 509

| | LC 509 Fibonacci | LC 70 Climb Stairs |
|---|---|---|
| Seed values | `dp = [0, 1]` → F(0), F(1) | `dp = [1, 2]` → ways(1), ways(2) |
| Loop range | `range(2, n+1)` | `range(3, n+1)` |
| Sequence | 0, 1, 1, 2, 3, 5 … | 1, 2, 3, 5, 8 … |
| Relationship | — | `ways(n) = F(n+1)` |

The two solutions are **structurally identical** — only the seed values differ.

---

## Trace Table — `n = 5`

| `i` | `dp[0]` (ways(i-1)) | `dp[1]` (ways(i)) | `total` |
|---|---|---|---|
| — | 1 | 2 | — *(initial state)* |
| 3 | 1 | 2 | **3** |
| 4 | 2 | 3 | **5** |
| 5 | 3 | 5 | **8** ✅ |

There are **8** distinct ways to climb 5 stairs.

---

## Why `range(3, n+1)`?

Same reasoning as LC 509:
- **Start at `3`** — `ways(1)` and `ways(2)` are already pre-seeded in `dp`, so we begin computing from step 3.
- **End at `n+1`** — Python's `range` is right-exclusive; `n+1` ensures the loop runs through `i = n`, computing `ways(n)`.

---

## Complexity Analysis

| | Complexity |
|---|---|
| **Time** | O(n) — single pass from 3 to n |
| **Space** | O(1) — only two variables, no full array |

---

## Key Takeaways

1. **Recognize the Fibonacci pattern** — whenever `f(n) = f(n-1) + f(n-2)`, think Fibonacci and apply the rolling window DP.
2. **Seed values determine the sequence** — the only difference from LC 509 is starting with `[1, 2]` instead of `[0, 1]`.
3. **Eliminate dead variables** — `dp = [0] * 3` with `dp[0]` unused is a red flag; always question whether every index is needed.
4. **Loop range follows the seed** — if you pre-seed 2 base cases (indices 1 and 2, or steps 1 and 2), your loop must start at the next value (3).
