# LC 746 · Min Cost Climbing Stairs

## Problem Statement

You are given an integer array `cost` where `cost[i]` is the cost of the `i`-th step.  
Once you pay the cost, you can climb **1 or 2 steps**.  
You can start from index `0` or index `1`.  
Return the **minimum cost** to reach the top of the floor (one step past the last index).

**Constraints:** `2 <= cost.length <= 1000`, `0 <= cost[i] <= 999`

---

## Key Insight — "Top" is One Step Beyond the Array

The top is **not** the last element of `cost` — it is an imaginary step at index `len(cost)`.  
This is why `dp` has size `len(cost) + 1`, and the answer is `dp[len(cost)]`.

```
cost = [10, 15, 20]
indices:  0    1    2    (3) ← top, not in cost array
```

You reach the top (index 3) by stepping from index 2 (paying cost[2]) or index 1 (paying cost[1]).

---

## DP Definition

```
dp[i] = minimum total cost to reach step i
```

- `dp[0] = 0` — start here for free
- `dp[1] = 0` — start here for free
- `dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])` for `i >= 2`

At each step `i`, you could have come from:
- Step `i-1` (paid `cost[i-1]`, took 1 step), or
- Step `i-2` (paid `cost[i-2]`, took 2 steps)

Take the cheaper option.

---

## Solution

```python
class Solution:
    def minCostClimbingStairs(self, cost: List[int]) -> int:
        dp = [0] * (len(cost) + 1)   # size n+1 to include the "top" step

        for i in range(2, len(cost) + 1):
            dp[i] = min(dp[i-1] + cost[i-1],   # arrive from one step below
                        dp[i-2] + cost[i-2])    # arrive from two steps below

        return dp[len(cost)]                    # cost to reach the top
```

---

## Comparison with LC 70 — Climb Stairs Family

| | LC 70 Climb Stairs | LC 746 Min Cost Climbing Stairs |
|---|---|---|
| Goal | Count distinct ways | Minimize total cost |
| Recurrence | `dp[i] = dp[i-1] + dp[i-2]` | `dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])` |
| Seed | `ways(1)=1, ways(2)=2` | `dp[0]=0, dp[1]=0` (free start) |
| "Top" position | Step `n` | Index `len(cost)` (beyond array) |
| Operation | Sum | Min |

Same rolling structure, different operation at each step.

## Trace Table — `cost = [10, 15, 20]`

`dp` has size 4: indices 0, 1, 2, 3 (index 3 = top)

| `i` | From `i-1` | From `i-2` | `dp[i]` |
|---|---|---|---|
| — | — | — | `dp = [0, 0, 0, 0]` *(initial)* |
| 2 | `dp[1] + cost[1]` = 0+15 = 15 | `dp[0] + cost[0]` = 0+10 = 10 | **10** |
| 3 | `dp[2] + cost[2]` = 10+20 = 30 | `dp[1] + cost[1]` = 0+15 = 15 | **15** ✅ |

Minimum cost to reach the top = **15**  
(Start at index 1, pay 15, jump 2 steps directly to the top.)

| 6  | 3+100=103 | 2+1=3     | **3**  |
| 7  | 3+1=4     | 3+100=103 | **4**  |
| 8  | 4+1=5     | 3+1=4     | **4**  |

## Why `range(2, len(cost) + 1)`?

- **Start at `2`** — `dp[0]` and `dp[1]` are both `0` (free starting positions), so computation begins at step 2.
- **End at `len(cost) + 1`** — the top is at index `len(cost)`, and Python's `range` is right-exclusive, so `+1` is needed to include that final step.

This is the same `range(start, n+1)` pattern from LC 509 and LC 70.

---

## Complexity Analysis

| | Complexity |
|---|---|
| **Time** | O(n) — single pass through all steps |
| **Space** | O(n) — full `dp` array of size `n+1` |

> **Space optimization:** Like LC 509/70, this can be reduced to O(1) using a two-variable rolling window since `dp[i]` only depends on `dp[i-1]` and `dp[i-2]`.

---

## Key Takeaways

1. **"Top" is beyond the array** — allocate `dp` of size `len(cost) + 1` and return `dp[len(cost)]`, not `dp[-1]`.
2. **Both starting positions are free** — seed `dp[0] = dp[1] = 0`; the cost is paid when you *leave* a step, not when you arrive.
3. **Same recurrence family as LC 70** — `dp[i]` depends on the previous two values; the only change is using `min(... + cost)` instead of a plain sum.
4. **Index alignment** — `dp[i]` represents the cost to *reach* step `i`, so the cost paid to get there is `cost[i-1]` (the step you just left), not `cost[i]`.
