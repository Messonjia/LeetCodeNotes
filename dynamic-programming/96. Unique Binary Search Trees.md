# LC 96 · Unique Binary Search Trees

## Problem Statement

Given an integer `n`, return the number of structurally unique BSTs which has exactly `n` nodes of unique values from `1` to `n`.

**Example:**
```
Input: n = 3
Output: 5
```

**Constraints:** `1 <= n <= 19`

---

## Core Insight

For any value `j` chosen as the **root** of the BST:
- The **left subtree** contains values `1` to `j-1` → `j-1` nodes
- The **right subtree** contains values `j+1` to `n` → `n-j` nodes

The number of unique BSTs with root `j` is:

```
dp[j-1] * dp[n-j]
```

Summing over all possible roots `j` from `1` to `n` gives `dp[n]`.

---

## DP Definition

| Term | Meaning |
|------|---------|
| `dp[i]` | Number of structurally unique BSTs with `i` nodes |
| `dp[0] = 1` | Base case: one empty tree exists |
| `dp[1] = 1` | Only one BST with a single node |

**Recurrence:**
```
dp[i] = sum of dp[j-1] * dp[i-j]  for j in range(1, i+1)
```

---

## Solution

```python
class Solution:
    def numTrees(self, n: int) -> int:
        dp = [0] * (n + 1)   # indices 0..n, need n+1 slots
        dp[0] = 1             # base case: empty tree
        for i in range(1, n + 1):
            for j in range(1, i + 1):   # j = root candidate
                dp[i] += dp[j - 1] * dp[i - j]
        return dp[n]
```

---

## Why `[0] * (n + 1)`?

We need valid indices `dp[0]` through `dp[n]` — that is `n + 1` slots.

```
dp = [0] * n        # indices 0..n-1  →  dp[n] causes IndexError ✗
dp = [0] * (n + 1)  # indices 0..n   →  dp[n] works fine         ✓
```

Think of it like building floors: to have a **Floor n**, you need an **(n+1)-floor building** (floors 0, 1, 2, …, n).

`dp[0] = 1` is not padding — it is a **meaningful base case** ("there is exactly one empty tree") that every multiplication in the recurrence depends on.

---

## Trace Table — `n = 3`

| i | j | dp[j-1] | dp[i-j] | contribution | dp[i] after |
|---|---|---------|---------|-------------|-------------|
| 1 | 1 | dp[0]=1 | dp[0]=1 | 1×1=1       | dp[1]=1     |
| 2 | 1 | dp[0]=1 | dp[1]=1 | 1×1=1       | 1           |
| 2 | 2 | dp[1]=1 | dp[0]=1 | 1×1=1       | dp[2]=2     |
| 3 | 1 | dp[0]=1 | dp[2]=2 | 1×2=2       | 2           |
| 3 | 2 | dp[1]=1 | dp[1]=1 | 1×1=1       | 3           |
| 3 | 3 | dp[2]=2 | dp[0]=1 | 2×1=2       | dp[3]=5     |

**Answer: `5`** ✓

---

## Complexity Analysis

| | Complexity |
|---|---|
| **Time** | O(n²) — double loop over `i` and `j` |
| **Space** | O(n) — the `dp` array of size `n+1` |

---

## Key Takeaways

1. **Root enumeration drives the recurrence** — fixing the root splits the problem into independent left and right subproblems whose counts multiply.
2. **`dp[0] = 1` is a real base case**, not a placeholder — the empty tree is a valid structure that anchors all multiplications.
3. **Always use `(n + 1)` slots** when your dp array needs indices `0` through `n`. This is the standard pattern for almost all 1D DP problems.
4. This sequence of answers (1, 1, 2, 5, 14, …) is the famous **Catalan numbers**, appearing in many combinatorics problems.
