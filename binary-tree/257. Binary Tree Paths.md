# Binary Tree Paths

## Problem

Given the root of a binary tree, return **all root-to-leaf paths** in any order.

> A **leaf** is a node with no children.

### Example

```
    1
   / \
  2   3
   \
    5
```

**Output:** `["1->2->5", "1->3"]`

---

## Approach: DFS + Backtracking

Use **Depth-First Search** with a shared `path` list. At each node:
1. **Choose** — append the current node to the path
2. **Explore** — recurse into left and right children
3. **Un-choose** — `pop()` the node off the path (backtrack)

### Why Backtracking?
The same `path` list is shared across all recursive calls. By calling `path.pop()` after exploring a node's subtree, we "undo" that node's contribution before moving to the next branch.

---

## Solution

```python
class Solution:
    def binaryTreePaths(self, root: Optional[TreeNode]) -> List[str]:
        result = []

        def dfs(node, path):
            if not node:
                return

            path.append(str(node.val))          # 1. CHOOSE

            if not node.left and not node.right:
                result.append("->".join(path))  # 2. RECORD leaf path
            else:
                dfs(node.left, path)            # 3. EXPLORE left
                dfs(node.right, path)           # 4. EXPLORE right

            path.pop()                          # 5. UN-CHOOSE (backtrack)

        dfs(root, [])
        return result
```

---

## Step-by-Step Trace

| Call | Node | Action | Path After |
|------|------|--------|------------|
| 1 | 1 | append "1" | `["1"]` |
| 2 | 2 | append "2" | `["1","2"]` |
| 3 | None | return | `["1","2"]` |
| 4 | 5 | append "5", **leaf** → record `"1->2->5"`, pop | `["1","2"]` |
| — | back to 2 | pop | `["1"]` |
| 5 | 3 | append "3", **leaf** → record `"1->3"`, pop | `["1"]` |
| — | back to 1 | pop | `[]` |

---

## Complexity

| | Backtracking | Pass Copies |
|---|---|---|
| **Time** | O(N) | O(N) |
| **Space** | O(H) — reuses one list | O(N) — new string per call |
| **Explicit undo?** | ✅ `path.pop()` | ❌ immutable strings |

> **H** = height of tree, **N** = number of nodes

---

## Common Mistake

```python
path.pop   # ❌ only references the method — does NOT call it
path.pop() # ✅ actually calls it and removes the last element
```

Without `()`, backtracking never happens and `path` keeps growing, producing incorrect results.

---

## Key Takeaway

> The **Choose → Explore → Un-choose** pattern is the backbone of backtracking.  
> Always use `path.pop()` (with parentheses!) to restore state after recursion.
