# 112. Path Sum

## Problem Description

Given the `root` of a binary tree and an integer `targetSum`, return `true` if the tree has a **root-to-leaf path** such that the sum of all values along the path equals `targetSum`.

> A **leaf** is a node with no children.

---

## Examples

```
Tree:           targetSum = 22
       5
      / \
     4   8
    /   / \
   11  13   4
  /  \       \
 7    2        1

Path: 5 → 4 → 11 → 2 = 22  ✅  → return True
```

---

## Approach: DFS Backtracking (Recursive)

### Core Idea

Start from the root and subtract each node's value from `targetSum` as we traverse downward. When we reach a **leaf node**, check if the remaining count equals `0`.

### Key Steps

1. **Base case — empty tree:** If `root` is `None`, return `False`.
2. **Initialize traversal:** Subtract `root.val` from `targetSum` before calling the helper.
3. **Leaf check:** At a leaf node (no left or right child), return `True` if `count == 0`.
4. **Recurse left/right:** For each child, subtract the child's value before recursing, and restore (backtrack) if the path doesn't succeed.

### Why Backtracking?

The count variable is mutated before each recursive call and restored afterward. This avoids creating new variables at each level and keeps space usage minimal.

---

## Solution

```python
class Solution:
    def traversal(self, cur: TreeNode, count: int) -> bool:
        # Leaf node reached — check if remaining sum is 0
        if not cur.left and not cur.right and count == 0:
            return True
        if not cur.left and not cur.right:
            return False

        if cur.left:
            count -= cur.left.val
            if self.traversal(cur.left, count):
                return True
            count += cur.left.val  # backtrack

        if cur.right:
            count -= cur.right.val
            if self.traversal(cur.right, count):
                return True
            count += cur.right.val  # backtrack

        return False

    def hasPathSum(self, root: Optional[TreeNode], targetSum: int) -> bool:
        if root is None:
            return False
        return self.traversal(root, targetSum - root.val)
```

---

## Complexity Analysis

| | Complexity | Explanation |
|---|---|---|
| **Time** | O(N) | Every node is visited at most once |
| **Space** | O(H) | Call stack depth equals tree height H (O(log N) balanced, O(N) worst case) |

---

## Trace Example

For `targetSum = 22` on the tree above:

```
hasPathSum(root=5, target=22)
  → traversal(5, count=17)       # 22 - 5
    → traversal(4, count=13)     # 17 - 4
      → traversal(11, count=2)   # 13 - 11
        → traversal(7, count=-5) # 2 - 7, leaf, count ≠ 0 → False
        → traversal(2, count=0)  # 2 - 2, leaf, count == 0 → ✅ True
```

## Edge Cases

| Scenario | Expected Output |
|---|---|
| Empty tree (`root = None`) | `False` |
| Single node, value == targetSum | `True` |
| Single node, value ≠ targetSum | `False` |
| Negative values in tree | Handled correctly |

---

## Alternative: Cleaner Recursive Approach

A more concise version passes the remaining sum directly without manual backtracking:

```python
def hasPathSum(self, root: Optional[TreeNode], targetSum: int) -> bool:
    if not root:
        return False
    if not root.left and not root.right:
        return root.val == targetSum
    return (self.hasPathSum(root.left,  targetSum - root.val) or
            self.hasPathSum(root.right, targetSum - root.val))
```

Both approaches have identical time and space complexity. The alternative is more idiomatic Python.
