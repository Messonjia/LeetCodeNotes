# Find Bottom Left Tree Value

## Problem

Given the root of a binary tree, return the **leftmost value in the last row** of the tree.

### Example

```
        2
       / \
      1   3
```
**Output:** `1`

```
        1
       / \
      2   3
     / \   \
    4   5   6
       /
      7
```
**Output:** `7`

---

## Approach: DFS with Depth Tracking

Use DFS and track the **current depth** as we go deeper. Whenever we hit a leaf node that is **deeper than any previously seen leaf**, update the result.

### Why does this give the leftmost value?
Because we always explore the **left subtree before the right**. So for any given depth, the first leaf we encounter at that depth will always be the leftmost one. Later leaves at the same depth (from the right subtree) won't overwrite the result since we use strict `>` comparison.

---

## Solution

```python
class Solution:
    def findBottomLeftValue(self, root: Optional[TreeNode]) -> int:
        self.max_depth = float('-inf')
        self.result = None
        self.traversal(root, 0)
        return self.result

    def traversal(self, node, depth):
        if not node.left and not node.right:       # leaf node
            if depth > self.max_depth:             # deepest leaf seen so far
                self.max_depth = depth
                self.result = node.val             # update result
            return

        if node.left:
            depth += 1
            self.traversal(node.left, depth)       # explore left first
            depth -= 1
        if node.right:
            depth += 1
            self.traversal(node.right, depth)      # explore right second
            depth -= 1
```

## Step-by-Step Trace

Using this tree:
```
        1
       / \
      2   3
     / \   \
    4   5   6
       /
      7
```

| Call | Node | Depth | Action |
|------|------|-------|--------|
| 1 | 1 | 0 | has children, go left |
| 2 | 2 | 1 | has children, go left |
| 3 | 4 | 2 | leaf! depth 2 > -inf → result = **4**, max_depth = 2 |
| 4 | 5 | 2 | has children, go left |
| 5 | 7 | 3 | leaf! depth 3 > 2 → result = **7**, max_depth = 3 |
| 6 | 3 | 1 | has children, go right |
| 7 | 6 | 2 | leaf! depth 2 < 3 → no update |

**Final result: `7`** ✅

---

## Key Design Decisions

### 1. `depth += 1` / `depth -= 1` vs passing `depth + 1`
The solution manually increments and decrements `depth` around each recursive call instead of passing `depth + 1` directly. Both are correct, but passing `depth + 1` is cleaner and less error-prone:

```python
# Current approach — works but easy to make mistakes
depth += 1
self.traversal(node.left, depth)
depth -= 1

# Simpler equivalent
self.traversal(node.left, depth + 1)
```

### 2. Strict `>` keeps the leftmost value
```python
if depth > self.max_depth:   # ✅ strict — first (leftmost) leaf at a new depth wins
if depth >= self.max_depth:  # ❌ wrong — rightmost leaf at that depth would win
```

### 3. Left before right guarantees leftmost
```python
if node.left:
    self.traversal(node.left, depth + 1)   # always explored first
if node.right:
    self.traversal(node.right, depth + 1)  # only updates result if deeper
```

---

## Complexity

| | Value |
|---|---|
| **Time** | O(N) — every node is visited once |
| **Space** | O(H) — recursion stack, H = height of tree |

---

## Key Takeaway

> To find the **leftmost** value at the **deepest** level:  
> 1. Always recurse **left before right**  
> 2. Only update the result when you reach a **strictly deeper** leaf  
> 
> These two rules together guarantee the result is always the leftmost node at the maximum depth.
