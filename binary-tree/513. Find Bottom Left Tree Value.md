# Sum of Left Leaves

## Problem

Given the root of a binary tree, return the **sum of all left leaves**.

> A **leaf** is a node with no children.  
> A **left leaf** is a leaf that is the **left child** of another node.

### Example

```
    3
   / \
  9  20
    /  \
   15   7
```

**Output:** `24` (9 + 15)

---

## Approach: DFS Recursion

Recursively traverse the tree. The key distinction is:
- A left leaf must be **both a leaf AND a left child**
- So the check for a left leaf must be done **from the parent**, not the node itself — because only the parent knows if a node is a left child

### Logic
1. **Base case** — if node is `None`, return `0`
2. **Base case** — if node is itself a leaf (no children), return `0` (no left leaves below)
3. **Left subtree** — recurse into left child; if left child is a leaf, override with its value
4. **Right subtree** — recurse into right child (left leaves can exist deep in the right subtree)
5. Return `leftValue + rightValue`

---

## Solution

```python
class Solution:
    def sumOfLeftLeaves(self, root: Optional[TreeNode]) -> int:
        if root is None:
            return 0
        if root.left is None and root.right is None:
            return 0

        leftValue = self.sumOfLeftLeaves(root.left)
        if root.left and not root.left.left and not root.left.right:
            leftValue = root.left.val          # root.left is a left leaf — capture its value

        rightValue = self.sumOfLeftLeaves(root.right)

        return leftValue + rightValue
```

---

## Step-by-Step Trace

Using this tree:
```
    3
   / \
  9  20
    /  \
   15   7
```

| Call | Node | left leaf check | leftValue | rightValue | return |
|------|------|----------------|-----------|------------|--------|
| 1 | 3 | `9` is a leaf ✅ | 9 | `sumOfLeftLeaves(20)` | 9 + 15 = **24** |
| 2 | 20 | `15` is a leaf ✅ | 15 | `sumOfLeftLeaves(7)` | 15 + 0 = **15** |
| 3 | 7 | no children → base case | — | — | **0** |
| 4 | 9 | no children → base case | — | — | **0** |

---

## Why Check From the Parent?

```python
# ✅ Correct — parent checks if its left child is a leaf
if root.left and not root.left.left and not root.left.right:
    leftValue = root.left.val

# ❌ Wrong — a node cannot know it's a "left" child by itself
#    There's no way to tell from inside a node whether it's left or right
```

A node has no knowledge of its own position in the tree. Only its **parent** knows whether it is a left or right child, so the "is this a left leaf?" check must always happen one level up.

---

## Complexity

| | Value |
|---|---|
| **Time** | O(N) — every node is visited once |
| **Space** | O(H) — recursion stack, H = height of tree |

---

## Key Takeaway

> When a problem depends on a node's **relationship to its parent** (e.g. left vs right child),  
> perform the check **at the parent level**, not inside the node's own recursive call.
