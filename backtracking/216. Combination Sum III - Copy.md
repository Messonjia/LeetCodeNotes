## Approach: Backtracking with Pruning

### Core Idea

Use DFS/backtracking to build combinations incrementally. At each step, choose a number and recurse. When the path reaches length `k`, record it. On the way back, undo the last choice (backtrack).

### Pruning Optimization

A naive loop runs `for i in range(startIndex, n + 1)`, which explores starts that can never yield a full combination.

**Key insight:** if you start picking from index `i`, the longest remaining sequence is `[i, i+1, ..., n]` with length `n - i + 1`. For a valid combination, you need at least `k - len(path)` more elements, so:

```
n - i + 1 >= k - len(path)
=> i <= n - (k - len(path)) + 1
```

So the loop upper bound (exclusive) becomes `n - (k - len(path)) + 2`.

| `n=4, k=3` at root | Remaining | Valid? |
|---|---|---|
| `i = 1` | `[1,2,3,4]` — 4 elements | ✅ |
| `i = 2` | `[2,3,4]` — 3 elements | ✅ |
| `i = 3` | `[3,4]` — 2 elements | ❌ pruned |
| `i = 4` | `[4]` — 1 element | ❌ pruned |

## Final Correct Solution

```python
class Solution:
    def combine(self, n: int, k: int) -> List[List[int]]:
        result = []
        self.backtracking(n, k, 1, [], result)
        return result                          # ← must return result

    def backtracking(self, n, k, startIdx, path, result):
        if len(path) == k:
            result.append(path[:])             # snapshot of current path
            return
        for i in range(startIdx, n - (k - len(path)) + 2):  # pruned upper bound
            path.append(i)
            self.backtracking(n, k, i + 1, path, result)
            path.pop()                         # backtrack
```

## Complexity Analysis

| | Value |
|---|---|
| **Time** | O(C(n, k) · k) — C(n,k) combinations, each copied in O(k) |
| **Space** | O(k) recursion depth + O(C(n,k) · k) for output |

Pruning reduces the number of recursive calls but does not change the asymptotic complexity, since valid leaves dominate.

## Key Takeaways

- `result.append(path[:])` — always copy the path, not the reference; `path[:]` creates a snapshot.
- The backtracking helper mutates `path` in place; the caller (`combine`) owns `result` and must return it.
- Pruning the loop upper bound is a common optimization pattern in combination/subset backtracking problems.
- Always verify that the entry-point method **returns** the accumulated result, not just builds it.
