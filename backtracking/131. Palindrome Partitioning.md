# LeetCode Study Note
## 131. Palindrome Partitioning
`Medium` В· `Backtracking` В· `DFS` В· `Two Pointers` В· `String`

---

## 1. Problem Statement

Given a string `s`, partition `s` such that every substring of the partition is a palindrome. Return all possible palindrome partitionings of `s`.

**Example:**
```
Input:  s = "aab"
Output: [["a","a","b"], ["aa","b"]]
```

---

## 2. Key Insight & Strategy

This is a **backtracking + substring validation** problem. At each step, try every possible cut from the current position вЂ” if the resulting substring is a palindrome, add it to the path and recurse on the remainder. If we reach the end of the string, the full path is a valid partition.

**Two sub-problems to solve:**
- **Enumeration** вЂ” use backtracking to try all ways to cut the string
- **Validation** вЂ” check whether each candidate substring is a palindrome

---

## 3. Solution Code

```python
class Solution:
    def partition(self, s: str) -> List[List[str]]:
        result = []
        self.backtracking(s, 0, [], result)
        return result

    def backtracking(self, s, startIndex, path, result):
        # Base case: we've reached the end вЂ” all substrings in path are palindromes
        if startIndex == len(s):
            result.append(path[:])      # Append a COPY of path
            return

        for i in range(startIndex, len(s)):
            if self.isPalin(s, startIndex, i):          # Only recurse if palindrome
                path.append(s[startIndex : i + 1])      # Cut s[startIndex..i]
                self.backtracking(s, i + 1, path, result)  # Recurse on remainder
                path.pop()                               # Backtrack

    def isPalin(self, s: str, start: int, end: int) -> bool:
        i, j = start, end
        while i < j:
            if s[i] != s[j]:
                return False
            i += 1
            j -= 1          # в†ђ must move j, NOT i again (common bug!)
        return True
```

---

## 4. How the Cutting Works

`startIndex` marks the start of the current unprocessed portion of `s`. The loop variable `i` marks the **end** of the candidate substring. We test every possible end position:

```
s = "aab",  startIndex = 0

i=0 в†’ candidate = s[0:1] = "a"   в†’ palindrome вњ… в†’ recurse with startIndex=1
i=1 в†’ candidate = s[0:2] = "aa"  в†’ palindrome вњ… в†’ recurse with startIndex=2
i=2 в†’ candidate = s[0:3] = "aab" в†’ NOT palindrome вќЊ в†’ skip
```

At each recursive level, `startIndex` advances to `i + 1`, narrowing the remaining string.

---

## 5. Base Case вЂ” Why `startIndex == len(s)`?

When `startIndex` reaches the end of the string, there are no characters left to cut вЂ” every character has been assigned to a palindrome substring in `path`. This means the current path is a complete valid partition, so we record it.

```python
if startIndex == len(s):
    result.append(path[:])   # path[:] is a copy вЂ” path itself will be mutated later
    return
```

---

## 6. The `isPalin` Two-Pointer вЂ” Common Bug

The palindrome check uses two pointers moving **inward** from both ends:

```python
i, j = start, end
while i < j:
    if s[i] != s[j]:
        return False
    i += 1
    j -= 1    # вњ… CORRECT: move j left
```

**Easy mistake to make:**
```python
    i += 1
    i -= 1    # вќЊ WRONG: moves i right then immediately back вЂ” infinite loop!
```

If you only move `i` and never move `j`, the pointers never converge and the loop runs forever on any string with length > 1.

## 8. Complexity Analysis

| Metric | Value |
|--------|-------|
| Time | O(N В· 2^N) вЂ” 2^N possible partitions, O(N) to validate each |
| Space | O(N) вЂ” max recursion depth |

---

## 9. Key Takeaways

- **Base case is `startIndex == len(s)`** вЂ” not a sum check like Combination Sum; here we validate by exhausting the string
- **Only recurse when palindrome** вЂ” non-palindrome substrings are simply skipped (no `break` here, since a longer cut might still be valid)
- **`i + 1` when recursing** вЂ” each character belongs to exactly one partition segment, no reuse
- **`path[:]` is a copy** вЂ” always append a copy, not the list reference itself
- **`isPalin` two-pointer bug** вЂ” `j -= 1`, never `i -= 1`; both pointers must move inward each iteration
