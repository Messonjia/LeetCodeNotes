# LeetCode 51 · N-Queens

## Problem Statement

Place `n` queens on an `n × n` chessboard so that **no two queens attack each other** — meaning no two queens share the same row, column, or diagonal.

Return **all distinct board configurations**. Each solution is a list of strings where `'Q'` = queen, `'.'` = empty.

**Example (n = 4):**
```
Solution 1:      Solution 2:
. Q . .          . . Q .
. . . Q          Q . . .
Q . . .          . . . Q
. . Q .          . Q . .
```

---

## Core Insight: One Queen Per Row

Since queens attack entire rows, we can place **exactly one queen per row**. This reduces the problem to:

> For each row (0 → n-1), pick which **column** to place the queen — then validate it doesn't conflict with already-placed queens above.

This maps perfectly to **backtracking**: try each column, recurse to the next row, undo if we hit a dead end.

---

## How the Chessboard Is Built — Step by Step

### 1. Initialization

```python
chessboard = ['.' * n for _ in range(n)]
```

The board is a **list of `n` strings**, each of length `n`, all dots:

```
For n = 4:
chessboard = [
    '....',   # row 0
    '....',   # row 1
    '....',   # row 2
    '....'    # row 3
]
```

Each string is **immutable** in Python, so placing/removing a queen requires **string slicing**:

```python
# Place queen at (row, col):
chessboard[row] = chessboard[row][:col] + 'Q' + chessboard[row][col+1:]

# Remove queen (undo):
chessboard[row] = chessboard[row][:col] + '.' + chessboard[row][col+1:]
```

---

### 2. Backtracking Frame: `row` as the Decision Variable

```
backtracking(n, row=0, chessboard, result)
```

- **`row`** tells us which row we're currently filling.
- We iterate `col` from `0` to `n-1` and try placing a queen at `(row, col)`.
- Base case: when `row == n`, all rows are filled → record the solution.

```
Row 0: try col 0, 1, 2, 3  ← outermost choices
  Row 1: try col 0, 1, 2, 3  ← next level
    Row 2: try col 0, 1, 2, 3
      Row 3: try col 0, 1, 2, 3
        row == n → save solution  ✓
```

---

### 3. Validity Check: `isValid(row, col, chessboard)`

Before placing a queen at `(row, col)`, we check **three threat directions** (only looking **upward** — rows below are not yet filled):

#### ① Same Column (vertical threat)
```python
for i in range(row):
    if chessboard[i][col] == "Q":
        return False
```
Scan every row above in column `col`.

#### ② Upper-Left Diagonal (↖)
```python
i, j = row - 1, col - 1
while i >= 0 and j >= 0:
    if chessboard[i][j] == 'Q':
        return False
    i -= 1; j -= 1
```
Walk diagonally up-left until off the board.

#### ③ Upper-Right Diagonal (↗)
```python
i, j = row - 1, col + 1
while i >= 0 and j < len(chessboard):
    if chessboard[i][j] == 'Q':
        return False
    i -= 1; j += 1
```
Walk diagonally up-right until off the board.

> **Why not check row conflicts?** We place exactly one queen per row and move to the next row immediately — row conflicts are structurally impossible.

---

### 4. The Choose → Explore → Un-choose Pattern

```python
for col in range(n):
    if self.isValid(row, col, chessboard):          # guard
        chessboard[row] = ... + 'Q' + ...           # CHOOSE
        self.backtracking(n, row + 1, chessboard, result)  # EXPLORE
        chessboard[row] = ... + '.' + ...           # UN-CHOOSE
```

This is the classic backtracking template. The undo step restores the board so sibling branches see a clean state.

---

### 5. Recording a Solution

```python
if row == n:
    result.append(chessboard[:])
    return
```

`chessboard[:]` is a **shallow copy** of the list. Since each element is an immutable string, this correctly captures the current state before further mutations.

The final result is formatted in `solveNQueens`:
```python
return [[''.join(row) for row in solution] for solution in result]
```
(The strings are already joined, so `''.join(row)` is a no-op here — each `row` is already a string.)

## Complexity Analysis

| | Value | Reasoning |
|---|---|---|
| **Time** | O(n!) | At most n choices for row 0, n-1 for row 1, etc.; pruning reduces this significantly in practice |
| **Space** | O(n²) | The board itself; recursion stack is O(n) |

---

## Key Takeaways

| Concept | Application Here |
|---------|-----------------|
| **Backtracking** | Try a column, recurse, undo — classic Choose/Explore/Un-choose |
| **Row as recursion level** | One queen per row → `row` indexes both the decision level and the board row |
| **Immutable string workaround** | Use slicing to "mutate" a row string in Python |
| **Directional validity** | Only check upward — lower rows haven't been filled yet |
| **Shallow copy for snapshot** | `chessboard[:]` captures state when a leaf is reached |
