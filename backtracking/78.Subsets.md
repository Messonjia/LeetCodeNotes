# LeetCode 78 – Subsets

## Problem

Given an integer array `nums` of **unique** elements, return all possible subsets (the power set). The solution must not contain duplicate subsets.

**Example:**
```
Input:  nums = [1, 2, 3]
Output: [[], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]]
```

---

## Approach: Backtracking (Collect at Every Node)

### Core Idea

In most backtracking problems, we only collect results at the **leaf nodes** (e.g., when a path reaches a target length or sum). Subsets is different:

> **Every node in the recursion tree is a valid subset — collect `path` at every call, before the loop.**

This naturally captures subsets of all sizes: the empty set at the root, single-element sets at depth 1, pairs at depth 2, and so on.

### Why No Base Case?

There is no explicit base case needed. When `startIndex >= len(nums)`, the `for` loop simply doesn't execute and the function returns — the recursion terminates on its own.

---

## Solution

```python
from typing import List

class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        result = []
        path = []
        self.backtracking(nums, 0, path, result)
        return result

    def backtracking(self, nums, startIndex, path, result):
        result.append(path[:])          # collect current subset at every node
        for i in range(startIndex, len(nums)):
            path.append(nums[i])        # choose
            self.backtracking(nums, i + 1, path, result)  # explore
            path.pop()                  # undo
```

---

## Recursion Tree

Input: `nums = [1, 2, 3]`

```
[]                          ← collected (root)
├── [1]                     ← collected
│   ├── [1, 2]              ← collected
│   │   └── [1, 2, 3]      ← collected (leaf)
│   └── [1, 3]             ← collected (leaf)
├── [2]                     ← collected
│   └── [2, 3]             ← collected (leaf)
└── [3]                     ← collected (leaf)
```

**Total nodes = 8 = 2³ = 2^n** — one for every possible subset.

## Complexity

| | Complexity |
|---|---|
| **Time** | O(n · 2ⁿ) — 2ⁿ subsets, each copied in O(n) |
| **Space** | O(n) — recursion depth is at most n; result list excluded |

---

## Key Takeaways

- **Collect at every node, not just leaves** — this is the defining pattern that distinguishes subsets from path-sum or permutation problems
- **`startIndex` prevents duplicates** — passing `i + 1` ensures each element is only considered once and we never go backwards
- **`path[:]` is essential** — appending `path` directly would store a reference; as `path` mutates, all collected entries would end up the same. Always copy with `path[:]` (or `path.copy()`)
- **No explicit base case needed** — the loop naturally exhausts when `startIndex` reaches the end of `nums`

## Comparison: Subsets vs. Other Backtracking Problems

| Problem Type | When to Collect | Example |
|---|---|---|
| **Subsets** | Every node (before the loop) | LC 78 |
| **Combinations** | Only at leaves (path reaches target length) | LC 77 |
| **Path Sum** | Only at leaves (sum condition met) | LC 112 |
| **Permutations** | Only at leaves (path uses all elements) | LC 46 |
