# 332. Reconstruct Itinerary

## Problem Statement

Given a list of airline tickets `tickets[i] = [fromi, toi]`, reconstruct the itinerary in order. The itinerary must:
- Begin with `"JFK"`
- Use all tickets exactly once
- Among all valid itineraries, return the one with the **smallest lexical order**

---

## Core Insight: Eulerian Path on a Directed Graph

Model the problem as a **directed graph** where airports are nodes and tickets are edges. The task is to find an **Eulerian path** — a path that traverses every edge exactly once — starting from `"JFK"`.

The algorithm used is **Hierholzer's algorithm**, adapted with a post-order append trick.

---

## Construction Strategy

### Step 1 — Build the adjacency list
```python
targets = defaultdict(list)
for ticket in tickets:
    targets[ticket[0]].append(ticket[1])
```

### Step 2 — Sort destinations in reverse, use `pop()` for lexicographic greedy selection
```python
for key in targets:
    targets[key].sort(reverse=True)
```
Reverse sorting + `pop()` from the end = always pick the **smallest** destination next, with O(1) removal.

### Step 3 — DFS with post-order append
```python
def backtracking(airport, targets, result):
    while targets[airport]:
        next_airport = targets[airport].pop()
        backtracking(next_airport, targets, result)
    result.append(airport)   # append AFTER recursion returns
```

### Step 4 — Reverse the result
```python
return result[::-1]
```
Nodes are appended when they have no remaining outgoing edges (deepest/dead-end nodes first), so reversing gives the correct departure order.

---

## Step-by-Step Trace

**Input:** `[["JFK","ATL"],["JFK","SFO"],["ATL","SFO"],["ATL","LAX"],["SFO","JFK"],["LAX","SFO"]]`

**Sorted adjacency lists (reverse order):**
| Airport | List (reverse sorted) | `pop()` gives |
|---------|----------------------|---------------|
| JFK     | `[SFO, ATL]`         | ATL first     |
| ATL     | `[SFO, LAX]`         | LAX first     |
| LAX     | `[SFO]`              | SFO           |
| SFO     | `[JFK]`              | JFK           |

**DFS call order:**
```
backtracking(JFK)
  → backtracking(ATL)
      → backtracking(LAX)
          → backtracking(SFO)
              → backtracking(JFK)       # JFK has SFO left
                  → backtracking(SFO)   # SFO: dead end → append SFO
                append JFK
              append SFO
          append LAX
      append ATL
  append JFK
```

**result (before reverse):** `[SFO, JFK, SFO, LAX, ATL, JFK]`

**result (after reverse):** `[JFK, ATL, LAX, SFO, JFK, SFO]` ✓

---

## Why Post-Order Append Works

The key insight: **any dead-end node must appear later in the itinerary**, because it can only be reached after other branching paths are exhausted. By appending a node *after* its recursive call completes (post-order), dead ends accumulate at the front of the `result` list. Reversing corrects the order — no backtracking or path retrying needed.

This handles the subtle case where you must finish a sub-loop first before continuing the main path.

---

## Complexity Analysis

| | Complexity |
|---|---|
| **Time** | O(E log E) — sorting each adjacency list; E = number of tickets |
| **Space** | O(E) — adjacency list + recursion stack depth |

---

## Full Solution

```python
from collections import defaultdict

class Solution:
    def findItinerary(self, tickets):
        targets = defaultdict(list)
        for ticket in tickets:
            targets[ticket[0]].append(ticket[1])

        for key in targets:
            targets[key].sort(reverse=True)   # reverse sort for greedy pop()

        result = []
        self.backtracking("JFK", targets, result)
        return result[::-1]

    def backtracking(self, airport, targets, result):
        while targets[airport]:
            next_airport = targets[airport].pop()        # O(1), smallest first
            self.backtracking(next_airport, targets, result)
        result.append(airport)                           # post-order append
```

---

## Key Takeaways

- **Eulerian path problems** on directed graphs → think Hierholzer's algorithm
- **Reverse sort + `pop()`** is the standard pattern for lexicographic greedy DFS
- **Post-order append + reverse** elegantly handles dead ends without explicit backtracking — the call stack does the ordering work for you
- This same pattern applies to any problem requiring traversal of all edges exactly once with ordering constraints
