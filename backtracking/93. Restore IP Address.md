# LeetCode 93 – Restore IP Addresses

## Problem

Given a string `s` containing only digits, return all possible valid IP addresses that can be formed by inserting dots into `s`. You may not reorder or remove any digits.

A valid IP address has **exactly four segments**, each an integer in `[0, 255]` with **no leading zeros**.

---

## Approach: Backtracking

### Core Idea

Use DFS/backtracking to try placing dots after every valid segment. At each recursive call, we extend the current IP by one segment. When exactly **3 dots** have been placed (`pointNum == 3`), validate the remaining substring as the 4th segment.

### Key Parameters

| Parameter    | Meaning                                     |
|--------------|---------------------------------------------|
| `startIndex` | Where the next segment begins in `s`        |
| `pointNum`   | How many dots have been placed so far (0–3) |
| `current`    | The IP string built so far                  |
| `result`     | Collects all valid complete IP addresses    |

### Segment Validity (`isValid`)

A segment `s[start..end]` is valid if:
1. `start <= end` (non-empty)
2. No leading zero unless the segment is exactly `"0"`
3. The integer value is `<= 255`

---

## Solution

```python
from typing import List

class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        result = []
        self.backtracking(s, 0, 0, "", result)
        return result

    def backtracking(self, s, startIndex, pointNum, current, result):
        if pointNum == 3:
            # Validate and collect the final (4th) segment
            if self.isValid(s, startIndex, len(s) - 1):
                result.append(current + s[startIndex:])
            return

        for i in range(startIndex, len(s)):
            if self.isValid(s, startIndex, i):
                sub = s[startIndex:i + 1]
                self.backtracking(s, i + 1, pointNum + 1, current + sub + '.', result)
            else:
                break  # Early exit: longer segments won't be valid either

    def isValid(self, s, start, end):
        if start > end:
            return False
        if s[start] == '0' and start != end:   # leading zero
            return False
        num = 0
        for i in range(start, end + 1):         # iterate only [start, end]
            if not s[i].isdigit():              # check character, not whole string
                return False
            num = num * 10 + int(s[i])
            if num > 255:
                return False
        return True
```

## Trace Example

Input: `s = "25525511135"`

| Call depth | startIndex | pointNum | current            | Action                  |
|------------|------------|----------|--------------------|-------------------------|
| 1          | 0          | 0        | `""`               | Try seg `"2"`, `"25"`, `"255"` |
| 2          | 3          | 1        | `"255."`           | Try seg `"2"`, `"25"`   |
| 3          | 5          | 2        | `"255.25."`        | Try seg `"5"`, `"51"`, `"511"` |
| 4          | 6          | 3        | `"255.25.5."`      | Remaining `"11135"` → invalid (> 255) |
| 4          | 7          | 3        | `"255.25.51."`     | Remaining `"1135"` → invalid (> 255) |
| 4          | 8          | 3        | `"255.25.511."`    | `511 > 255` → isValid fails |
| ...        | ...        | ...      | ...                | Eventually finds valid ones |

**Output:** `["255.255.11.135", "255.255.111.35"]`

| | Complexity |
|---|---|
| **Time** | O(3⁴) = O(1) — at most 3 choices per segment × 4 segments; bounded constant |
| **Space** | O(n) — recursion depth is at most 4; result list excluded |

## My Bugs & Lessons Learned

### Bug 1 – Wrong loop range in `isValid`

```python
# ❌ Wrong: loops over entire string s, ignoring start/end bounds
for i in range(0, len(s)):

# ✅ Fix: loop only over the segment [start, end]
for i in range(start, end + 1):
```

`start` and `end` were passed in as bounds but completely ignored, causing the validator to check the whole string every time.

### Bug 2 – `.isdigit()` called on whole string, not a character

```python
# ❌ Wrong: checks if the entire string s is digits
if not s.isdigit():

# ✅ Fix: check the current character
if not s[i].isdigit():
```

This would always evaluate based on the full string `s` rather than the individual character at index `i`.

> **Note:** Since the problem guarantees `s` contains only digits, the `isdigit()` check is technically redundant here — but the indexing bug still caused incorrect behavior.

## Key Takeaways

- **Backtracking template:** choose segment → recurse → (implicitly undo by not mutating state)
- **Early `break`:** once a segment starting at `startIndex` is invalid at length `i - startIndex + 1`, longer ones will also be invalid → `break` prunes the tree
- **Base case at `pointNum == 3`:** don't recurse further — just validate the rest of `s` as the 4th segment in one shot
- Always double-check that helper functions use their **parameters**, not outer variables
