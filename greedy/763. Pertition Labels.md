# LC 763 · Partition Labels

## Problem

Given string `s`, partition it into as many parts as possible such that each letter appears in **at most one part**. Return a list of partition sizes.

**Example:**
```
Input:  s = "ababcbacadefegdehijhklij"
Output: [9, 7, 8]
```

---

## Core Insight

A character's partition **must** extend at least to its last occurrence. Greedily sweep left-to-right: maintain a window `[start, end]` where `end` always stretches to the furthest last-occurrence of any character seen so far. When the current index reaches `end`, the window is "closed" — no character inside appears further right.

---

## Annotated Solution

```python
class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        # Phase 1: build last-occurrence map  O(n) time, O(1) space (≤26 keys)
        last = {}
        for i, ch in enumerate(s):
            last[ch] = i          # overwrite → keeps only the last index

        # Phase 2: greedy window scan
        result, start, end = [], 0, 0
        for i, ch in enumerate(s):
            end = max(end, last[ch])  # extend window if this char reaches further
            if i == end:              # window sealed — all chars stay inside
                result.append(end - start + 1)
                start = i + 1         # next partition starts right after
        return result
```

---

## Trace Table — `s = "ababcc"`

Last-occurrence map: `{'a': 3, 'b': 4, 'c': 5}`

| i | ch | last[ch] | end (after max) | i == end? | result / start         |
|---|----|----------|-----------------|-----------|------------------------|
| 0 | a  | 3        | 3               | no        | —                      |
| 1 | b  | 4        | 4               | no        | —                      |
| 2 | a  | 3        | 4               | no        | —                      |
| 3 | b  | 4        | 4               | no        | —                      |
| **4** | **b** | **4** | **4**       | **yes ✓** | **append 5, start=5**  |
| **5** | **c** | **5** | **5**       | **yes ✓** | **append 1, start=6**  |

**Output:** `[5, 1]` → partitions `"ababc"` and `"c"`

## Complexity

| | |
|---|---|
| **Time** | O(n) — two linear passes |
| **Space** | O(1) — hash map has at most 26 keys |

---

## Key Takeaways

- **Greedy:** The window end is always the maximum last-occurrence of all characters seen so far. No backtracking needed.
- **Hash map (Phase 1):** Classic "last-index" map built in O(n). Keys are overwritten on each repeat — intentional.
- **Pattern — Greedy Interval Merging:** Keep expanding the right boundary until it equals the current index. Same idea appears in:
  - LC 56 · Merge Intervals
  - LC 45 · Jump Game II

---

## Common Pitfalls

- **Missing `start = i + 1`** after appending → off-by-one error in the next partition size.
- **Using `end = last[ch]` without `max`** → the window would shrink when a short-range character appears after a long-range one.
