# LC 122 · Best Time to Buy and Sell Stock II

## Problem Statement

Given an integer array `prices` where `prices[i]` is the stock price on day `i`, find and return the **maximum profit** you can achieve.

**Rules:**
- You may buy and/or sell on the same day.
- You can hold **at most one share** at a time.
- You may transact **as many times as you like**.

**Example:**
```
Input:  prices = [7, 1, 5, 3, 6, 4]
Output: 7
Explanation: Buy day 1 (price=1), sell day 2 (price=5), profit = 4.
             Buy day 3 (price=3), sell day 4 (price=6), profit = 3.
             Total profit = 7.
```

**Constraints:**
- `1 <= prices.length <= 3 * 10^4`
- `0 <= prices[i] <= 10^4`

---

## Greedy Insight — Why It Works Fast

### The Core Observation

Because we can transact unlimited times, there is no penalty for buying and selling on consecutive days. This means **any upward day-to-day move is independently capturable**.

Consider the math for any rising segment from day `i` to day `j`:

```
profit(i→j) = prices[j] - prices[i]
            = (prices[i+1] - prices[i])
            + (prices[i+2] - prices[i+1])
            + ...
            + (prices[j]   - prices[j-1])
```

A multi-day gain **decomposes exactly** into the sum of every positive daily gap. So instead of finding the optimal buy/sell windows, we can simply accumulate every positive `prices[i] - prices[i-1]`.

### Why Greedy Is Safe Here

- **Taking every positive gap** is equivalent to making the optimal trades — we never miss profit.
- **Skipping a negative gap** is always correct — holding through a down day only reduces our total.
- There are no transaction costs, cooldown periods, or holding limits that would make it suboptimal to capture small consecutive gains.

```
prices:  7    1    5    3    6    4
diff:      -6   +4   -2   +3   -2
                 ↑         ↑
              collect    collect    → total = 7
```

## Annotated Solution

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        result = 0                              # 累积总利润

        for i in range(1, len(prices)):
            earning = prices[i] - prices[i-1]  # 相邻两天的差值
            if earning > 0:                     # 只收割正向收益
                result += earning               # 贪心：每个上涨区间都累加

        return result
```

> **Key design choice:** The loop starts at index `1` and only looks one step back. We never need to track a `buy_price` variable — consecutive positive differences do the heavy lifting automatically.

## The "Valley-Peak" Equivalence

Another way to think about this: the greedy daily-diff approach is mathematically equivalent to the valley-peak strategy.

```
Valley-peak:  buy at valley 1 (price=1), sell at peak 5 (price=5)  → +4
              buy at valley 3 (price=3), sell at peak 6 (price=6)  → +3
              total = 7

Daily diff:   skip -6, take +4, skip -2, take +3, skip -2
              total = 7
```

They always produce the same result because summing consecutive positive diffs is identical to computing `peak - valley` over any rising window. The daily-diff version is simply easier to code.

## Key Takeaways

1. **Unlimited transactions = capture every positive daily move.** There's no reason to hold through a down day.
2. **The sum of consecutive diffs equals the total range gain.** `prices[j] - prices[i] = Σ(prices[k] - prices[k-1])` for k from i+1 to j. This is why the greedy decomposition is exact, not approximate.
3. **No buy-price tracking needed.** Unlike LC 121 (single transaction), we never need to remember where we bought because we're always looking at just the next day's gap.
4. **`earning > 0` is a strict inequality.** A flat day (`earning == 0`) contributes nothing and is correctly skipped.
5. This problem is intentionally easier than LC 121 (one transaction) or LC 123 (two transactions) — the unlimited transactions rule is what makes the greedy decomposition valid.

---

## Contrast with LC 121 (Single Transaction)

| Feature | LC 121 | LC 122 |
|---|---|---|
| Max transactions | 1 | Unlimited |
| Strategy | Track running minimum buy price | Sum all positive daily gaps |
| Algorithm | One-pass min tracking | One-pass greedy accumulation |
| Greedy valid? | No — must find best single window | Yes — every positive gap is independent |

In LC 121, we must find the single best `[buy, sell]` pair. In LC 122, the unlimited transactions rule breaks the problem into independent subproblems — one per consecutive day pair — so greedy works perfectly.

---

## Common Pitfalls

| Bug | Symptom | Fix |
|---|---|---|
| `earning >= 0` instead of `> 0` | Slightly wrong on flat segments; no bug in practice but conceptually wrong | Use strict `> 0` |
| Starting loop at `i = 0` | Index error: `prices[i-1]` → `prices[-1]` = last element in Python | Start at `i = 1` |
| Trying to track buy/sell days explicitly | Overcomplicates the solution | Trust the daily-diff decomposition — no explicit buy/sell tracking needed |
| Confusing with LC 121 | Using min-tracking instead of accumulation | Remember: unlimited transactions → accumulate all positive diffs |
