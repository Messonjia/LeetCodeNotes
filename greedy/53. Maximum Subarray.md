# LC 376 · Wiggle Subsequence

## Problem Statement

Given an integer array `nums`, return the **length of the longest wiggle subsequence**.

A **wiggle sequence** is one where the differences between successive elements strictly alternate between positive and negative. A sequence of length 1 or 2 (with unequal elements) is trivially a wiggle sequence.

**Example:**
```
Input:  nums = [1, 7, 4, 9, 2, 5]
Output: 6
```
Differences: `(+6, -3, +5, -7, +3)` — all alternating ✓

**Constraints:**
- `1 <= nums.length <= 1000`
- `0 <= nums[i] <= 1000`

---

## Greedy Insight — Why It Works Fast

The key question is: **do we need to try all possible subsequences?** No — and here's why.

### The Local Observation

Imagine scanning the array left to right. Every time the direction of change **flips** (up → down, or down → up), we've hit a **peak or valley**. Greedily counting every such flip gives the longest wiggle subsequence.

```
nums:  1    7    4    9    2    5
diffs:    +6   -3   +5   -7   +3
           ↑    ↑    ↑    ↑    ↑
         flip  flip flip flip flip  → 5 flips + 1 start = 6
```

### Why Greedy Is Safe Here

Suppose we **skip** a local peak (e.g., skip `7` above). We'd lose a valley-to-peak transition and gain nothing — the subsequence can only get shorter or stay the same. Therefore, including every peak and valley is **always at least as good** as skipping any of them. This greedy choice is locally optimal and globally optimal.

### Handling Flat Runs

For a flat segment (`nums[i] == nums[i+1]`), `curdiff = 0`. The condition:
```python
prediff <= 0 and curdiff > 0   # rising start counts
prediff >= 0 and curdiff < 0   # falling start counts
```
A flat run keeps `prediff` unchanged because we **only update `prediff` when we count**. This effectively ignores plateaus without any special-case logic.

## Annotated Solution

```python
class Solution:
    def wiggleMaxLength(self, nums: List[int]) -> int:
        if len(nums) <= 1:
            return len(nums)            # 边界：0 或 1 个元素直接返回

        curdiff = 0
        prediff = 0
        result = 1                      # 第一个元素默认算在结果里

        for i in range(len(nums) - 1):
            curdiff = nums[i+1] - nums[i]   # 计算相邻差值

            # 满足"摆动"条件：方向发生翻转
            if (prediff <= 0 and curdiff > 0) or \
               (prediff >= 0 and curdiff < 0):
                result += 1             # 贪心：遇到拐点就计数
                prediff = curdiff       # 只在拐点更新 prediff，忽略平坦段

        return result
```

> **Key design choice:** `prediff` is updated **only when we count a flip**. This is what makes flat runs transparent — `prediff` stays at its last meaningful value so the next real direction change is still detected correctly.

## Flat Run Edge Case

`nums = [1, 2, 2, 2, 3, 1]`

| i | curdiff | prediff | Condition met? | result |
|---|---------|---------|----------------|--------|
| 0 | +1      | 0       | `0<=0 and 1>0` ✓ | 2    |
| 1 | 0       | +1      | no (0 is not negative) | 2 |
| 2 | 0       | +1      | no             | 2      |
| 3 | +1      | +1      | no (same direction) | 2 |
| 4 | −2      | +1      | `1>=0 and -2<0` ✓ | 3   |

**Result: 3** — subsequence `[1, 3, 1]` or `[2, 3, 1]` ✓

## Complexity Analysis

| | Value | Reason |
|---|---|---|
| **Time** | O(n) | Single left-to-right pass |
| **Space** | O(1) | Only `curdiff`, `prediff`, `result` variables |

Compare with brute force (all subsequences): O(2ⁿ), or DP: O(n) time but O(n) space with a `up[]`/`down[]` array. Greedy wins on space.

---

## Key Takeaways

1. **Greedy = count every direction flip.** You never benefit from skipping a peak or valley.
2. **Only update `prediff` on a flip.** This is what silently absorbs flat runs.
3. **`prediff <= 0` (not `< 0`)** allows the very first element to pair with a rise OR a fall — handles the case where the sequence starts flat then moves.
4. **result starts at 1** because the first element is always in the subsequence.
5. This greedy is equivalent to a 2-state DP (`up` / `down`) but achieves O(1) space by collapsing the states into a single `prediff` variable.

---

## Common Pitfalls

| Bug | Example | Fix |
|---|---|---|
| Update `prediff` on every step | Flat run `[1,2,2,3]` overcounts | Only update `prediff = curdiff` inside the `if` block |
| Use `< 0` instead of `<= 0` | `[0, 1]` returns 1 instead of 2 | Use `<=` so `prediff=0` (start state) triggers on first flip |
| Start `result = 0` | Off-by-one on all inputs | The first element is always part of the answer → start at 1 |
