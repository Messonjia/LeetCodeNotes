# LeetCode 17 — Letter Combinations of a Phone Number

## Problem Statement

Given a string containing digits from `2–9`, return all possible letter combinations the number could represent (like old telephone buttons). Return the answer in any order.

**Digit → Letter Mapping:**
```
2 → abc    3 → def    4 → ghi    5 → jkl
6 → mno    7 → pqrs   8 → tuv    9 → wxyz
```

**Example:**
```
Input:  digits = "23"
Output: ["ad","ae","af","bd","be","bf","cd","ce","cf"]
```

---

## Key Insight — Different from Previous Combination Problems

In LC 77 and LC 216, we picked numbers from a **single pool** and used `startIndex` to avoid reuse.  
Here, each digit maps to a **different set of letters**, and we pick **one letter per digit** in sequence. There is no `startIndex` — instead we use `index` to track **which digit we are currently processing**.

```
digits = "23"

Level 0 (digit '2', letters "abc"):   pick a, b, or c
Level 1 (digit '3', letters "def"):   pick d, e, or f
                                              ↓
                             one full path = one combination
```

The recursion tree has **depth = len(digits)**, and each node branches by the number of letters on that digit's key.

---

## Approach: Backtracking

### Steps

1. Map each digit to its letter string via a lookup table.
2. Use `index` to track which digit is being processed (replaces `startIndex`).
3. At each level, iterate over all letters for `digits[index]`, append the letter, recurse to the next digit, then undo.
4. Base case: when `index == len(digits)`, the path is complete — record it.

### Why no `startIndex` here?

`startIndex` prevents re-picking from the **same pool**. Here each level draws from a **different pool** (a different digit's letters), so there's nothing to skip — we always iterate the full letter set for the current digit.

## Solution

```python
class Solution:
    def __init__(self):
        self.letterMap = [
            "",     # 0
            "",     # 1
            "abc",  # 2
            "def",  # 3
            "ghi",  # 4
            "jkl",  # 5
            "mno",  # 6
            "pqrs", # 7
            "tuv",  # 8
            "wxyz"  # 9
        ]
        self.result = []
        self.s = ""

    def backtracking(self, digits, index):
        if index == len(digits):        # all digits processed → combination complete
            self.result.append(self.s)
            return
        digit = int(digits[index])      # current digit
        letters = self.letterMap[digit] # its mapped letters
        for i in range(len(letters)):
            self.s += letters[i]                    # choose letter
            self.backtracking(digits, index + 1)    # recurse to next digit
            self.s = self.s[:-1]                    # backtrack: remove last char

    def letterCombinations(self, digits: str) -> List[str]:
        if len(digits) == 0:
            return []                   # edge case: empty input
        self.backtracking(digits, 0)
        return self.result
```

---

## Recursion Trace — `digits = "23"`

```
backtracking(index=0, s="")        ← digit '2', letters "abc"
│
├── s="a" → backtracking(index=1)  ← digit '3', letters "def"
│           ├── s="ad" → index==2, append "ad" ✅
│           ├── s="ae" → index==2, append "ae" ✅
│           └── s="af" → index==2, append "af" ✅
│
├── s="b" → backtracking(index=1)
│           ├── s="bd" → append "bd" ✅
│           ├── s="be" → append "be" ✅
│           └── s="bf" → append "bf" ✅
│
└── s="c" → backtracking(index=1)
            ├── s="cd" → append "cd" ✅
            ├── s="ce" → append "ce" ✅
            └── s="cf" → append "cf" ✅

Result: ["ad","ae","af","bd","be","bf","cd","ce","cf"]
```

## Complexity Analysis

| | Value |
|---|---|
| **Time** | O(4^n · n) — at most 4 letters per digit, n digits, each path copied in O(n) |
| **Space** | O(n) recursion depth + O(4^n · n) for output |

## Comparison with Previous Backtracking Problems

| | LC 77 Combinations | LC 216 Combo Sum III | LC 17 Phone Letters |
|---|---|---|---|
| Pool | Single (`1–n`) | Single (`1–9`) | **Multiple** (one per digit) |
| Navigation | `startIndex` | `startIndex` | **`index`** (digit position) |
| Pruning | Loop upper bound | Sum exceeded + loop bound | None needed |
| Path stored as | `List[int]` | `List[int]` | `str` |

## Key Takeaways

- When each recursive level draws from a **different source**, use a positional `index` instead of `startIndex`.
- The backtrack on a string is `self.s = self.s[:-1]` — remove the last character, mirroring `self.s += letters[i]`.
- Always handle the **empty input edge case** explicitly — `letterCombinations("")` should return `[]`, not `[""]`.
- Instance variables (`self.result`, `self.s`) work fine but couple state to the object; passing them as parameters is more reusable for interview settings.
