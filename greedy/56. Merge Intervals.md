# LC 738 · Monotone Increasing Digits

## Problem

An integer has **monotone increasing digits** if and only if each pair of adjacent digits `x` and `y` satisfies `x <= y`.

Given an integer `n`, return the **largest number ≤ n** with monotone increasing digits.

**Example 1:**
```
Input:  n = 10
Output: 9
```

**Example 2:**
```
Input:  n = 1234
Output: 1234   (already monotone increasing)
```

**Example 3:**
```
Input:  n = 332
Output: 299
```

---

## Core Insight

Scan from **right to left**. When a violation is found (`digits[i-1] > digits[i]`), decrement `digits[i-1]` by 1 and fill everything to the right with `'9'` — this maximizes the remaining digits while fixing the violation.

Why right-to-left? Decrementing a digit may create a new violation with its left neighbor. Scanning right-to-left lets each fix propagate naturally leftward without a second pass.

---

## Annotated Solution

```python
class Solution:
    def monotoneIncreasingDigits(self, n: int) -> int:
        strNum = list(str(n))          # convert to list of char strings

        for i in range(len(strNum) - 1, 0, -1):   # scan right to left
            if strNum[i - 1] > strNum[i]:          # violation found
                strNum[i - 1] = str(int(strNum[i - 1]) - 1)  # decrement left digit
                for j in range(i, len(strNum)):    # fill everything right with '9'
                    strNum[j] = '9'                # must be str, not int

        return int(''.join(strNum))
```

## Trace Table — `n = 332`

Initial: `strNum = ['3', '3', '2']`

| i | strNum[i-1] | strNum[i] | violation? | action                          | strNum after         |
|---|-------------|-----------|------------|---------------------------------|----------------------|
| 2 | '3'         | '2'       | 3 > 2 ✓   | decrement index 1 → '2', fill index 2 with '9' | `['3', '2', '9']` |
| 1 | '3'         | '2'       | 3 > 2 ✓   | decrement index 0 → '2', fill index 1..2 with '9' | `['2', '9', '9']` |

**Output:** `int('299')` → `299`

## Complexity

|           |                                                        |
|-----------|--------------------------------------------------------|
| **Time**  | O(n) — where n is the number of digits (at most 10 for 32-bit int) |
| **Space** | O(n) — for the digit list                             |

---

## Key Takeaways

- **Right-to-left greedy:** Fixing violations from right to left lets each decrement naturally cascade to the left without re-scanning.
- **Fill right with `'9'`:** After decrementing a digit, all digits to its right must become `'9'` to maximize the result.
- **String list, not int list:** `list(str(n))` creates a list of character strings. Every assignment — including the `'9'` fill — must use strings, or `''.join()` will raise a `TypeError`.

---

## Common Bugs

| Bug | Symptom | Fix |
|-----|---------|-----|
| `strNum[1-1]` instead of `strNum[i-1]` | Always accesses index 0, ignores loop variable | Use `strNum[i-1]` |
| `str(int(strNum[i-1] - 1))` — subtract before converting | `TypeError: unsupported operand type(s) for -: 'str' and 'int'` | Convert first: `str(int(strNum[i-1]) - 1)` |
| `len(NumStr)` instead of `len(strNum)` | `NameError: name 'NumStr' is not defined` | Use `len(strNum)` |
| `strNum[j] = 9` (int) instead of `'9'` (str) | `TypeError: sequence item N: expected str instance, int found` | Use `strNum[j] = '9'` |
