# LC 452 · Minimum Number of Arrows to Burst Balloons

## Problem Statement

Balloons are represented as intervals `[xstart, xend]` on a number line. An arrow shot at position `x` bursts every balloon where `xstart <= x <= xend`. Arrows travel upward infinitely.

Return the **minimum number of arrows** needed to burst all balloons.

---

## Core Insight: Greedy Interval Merging

This is a classic **greedy interval overlap** problem. The key observation:

- If two balloons **overlap**, one arrow can burst both.
- The goal is to maximize how many balloons each arrow can hit — i.e., find the largest groups of mutually overlapping balloons.
- Sort by `xstart`, then scan. When a new balloon overlaps the current group, **shrink the active window** to the intersection. When it doesn't overlap at all, fire an arrow and start a new group.

The shrinking step (`points[i][1] = min(prev_end, curr_end)`) is the heart of the algorithm — it tracks the tightest x-range that still hits all balloons in the current group.

---

## Solution

```python
from typing import List

class Solution:
    def findMinArrowShots(self, points: List[List[int]]) -> int:
        if not points:
            return 0

        points.sort(key=lambda x: x[0])   # sort by xstart
        ans = 1                             # at least one arrow needed

        for i in range(1, len(points)):
            if points[i - 1][1] < points[i][0]:
                # gap between balloons — current arrow can't reach, fire a new one
                ans += 1
            else:
                # overlap — shrink the window to the intersection
                points[i][1] = min(points[i - 1][1], points[i][1])

        return ans
```

---

## The Shrinking Window Explained

When balloons overlap, we update `points[i][1]` to be the **minimum of the two right edges**. This effectively replaces the current balloon's right boundary with the tightest constraint seen so far.

```
A: [1 -------- 6]
B:      [3 ---------- 9]
                 → overlap window: [3, 6]   (min right edge = 6)

C:                  [7 -------- 10]
                 → C starts at 7, window ends at 6 → no overlap → new arrow
```

By carrying the minimum right edge forward into `points[i][1]`, each subsequent comparison automatically checks against the entire group's constraint, not just the previous balloon.

**Why mutate `points[i][1]` instead of a separate variable?**  
Because the next iteration uses `points[i][1]` as `points[i-1][1]`. Storing the window's right edge directly in the array means no extra variable is needed — the forward propagation happens naturally through the loop.

---

## Trace Table

**Input:** `[[10,16],[2,8],[1,6],[7,12]]`

After sort by `xstart`: `[[1,6],[2,8],[7,12],[10,16]]`

| i | prev `[xstart,xend]` | curr `[xstart,xend]` | overlap? | action | ans |
|---|---|---|---|---|---|
| 1 | [1, **6**] | [2, 8] | yes (6 >= 2) | `points[1][1] = min(6,8) = 6` → curr becomes [2,**6**] | 1 |
| 2 | [2, **6**] | [7, 12] | no (6 < 7) | gap → new arrow | 2 |
| 3 | [7, **12**] | [10, 16] | yes (12 >= 10) | `points[3][1] = min(12,16) = 12` → curr becomes [10,**12**] | 2 |

**Result: 2** ✓

Arrow 1 can be shot anywhere in `[3, 6]` — hits `[1,6]` and `[2,8]`.  
Arrow 2 can be shot anywhere in `[10, 12]` — hits `[7,12]` and `[10,16]`.

---

## Edge Cases

| Case | Behavior |
|---|---|
| Empty input | Return 0 immediately |
| All balloons overlap | One arrow suffices; window keeps shrinking |
| No balloons overlap | Every balloon needs its own arrow; `ans` increments each iteration |
| Touching boundaries `[1,5]` and `[5,9]` | `5 >= 5` → **overlap** — one arrow at x=5 bursts both |

The touching-boundary case is important: the condition for no overlap is `prev_end < curr_start` (strictly less than), not `<=`. An arrow at x=5 satisfies both `x <= 5` and `x >= 5`.

## Complexity Analysis

| | Complexity |
|---|---|
| **Time** | O(n log n) — dominated by sorting |
| **Space** | O(1) extra — sorting is in-place; we mutate `points` directly |

---

## Key Takeaways

1. **Sort by start, track the tightest right edge** — the greedy invariant: at each step, we know the narrowest window where a single arrow can still hit all balloons seen so far.
2. **Mutating `points[i][1]`** propagates the window constraint forward without needing a separate variable.
3. **Strictly less than for gap detection**: `prev_end < curr_start` means no overlap. If they are equal (`prev_end == curr_start`), the boundaries touch and one arrow still works.
4. This pattern — sort intervals, scan and merge greedily — appears across many interval problems: LC 435 Non-overlapping Intervals, LC 56 Merge Intervals, LC 57 Insert Interval. The difference is usually what you count (gaps vs merges) and whether endpoints are inclusive.
5. **Equivalent reformulation:** the minimum number of arrows = the number of non-overlapping groups after greedy merging = total balloons minus the maximum number you can "merge" into groups. This is essentially the same as LC 435 (minimum intervals to remove to make the rest non-overlapping).
