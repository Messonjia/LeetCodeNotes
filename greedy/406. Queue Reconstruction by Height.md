# LC 406 · Queue Reconstruction by Height

## Problem Statement

Given an array `people` where `people[i] = [hi, ki]` represents a person of height `hi` with exactly `ki` people in front of them who have height **greater than or equal to** `hi`.

Reconstruct and return the queue.

---

## Core Insight: Sort Then Insert

The trick is to break the problem into two independent steps:

1. **Sort tallest first** — so that when you place a person, everyone already in the queue is taller or equal.
2. **Insert at index k** — since all existing people are taller, inserting at position `k` guarantees exactly `k` taller-or-equal people are in front. Shorter people added later don't disturb this count.

This works because **shorter people are invisible to taller people's `k` constraint**. A short person inserted anywhere in front of a tall person doesn't count toward that tall person's `k` value.

---

## Solution

```python
from typing import List

class Solution:
    def reconstructQueue(self, people: List[List[int]]) -> List[List[int]]:
        # Step 1: sort by height descending; ties broken by k ascending
        people.sort(key=lambda x: (-x[0], x[1]))

        # Step 2: greedily insert each person at index k
        que = []
        for p in people:
            que.insert(p[1], p)   # insert p at position p[1]
        return que
```

---

## `list.insert(index, value)` Explained

`list.insert(i, v)` inserts value `v` at position `i`, shifting everything at index `i` and beyond one step to the right.

```python
que = [10, 20, 30]
que.insert(1, 99)
# → [10, 99, 20, 30]
#         ↑ inserted here; 20 and 30 shifted right
```

So `que.insert(p[1], p)` places person `p` at exactly the position their `k` value demands.

---

## Why Sort by `(-h, k)`?

| Sort key | Purpose |
|---|---|
| `-x[0]` (height descending) | Ensures all people already in `que` are taller or equal when inserting the current person |
| `x[1]` (k ascending, for ties) | Among people of the same height, insert the one with smaller `k` first so later insertions at larger `k` land in the right spot |

**Why ties matter:** if two people share the same height, say `[7,0]` and `[7,1]`, you must insert `[7,0]` first. If you inserted `[7,1]` first (into an empty list at index 1 — which would fail or go to index 0), then inserted `[7,0]` at index 0, `[7,1]` would end up with `[7,0]` in front of it — giving it 0 taller-or-equal people in front instead of 1.

## Trace Table

**Input:** `[[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]`

After sort: `[[7,0],[7,1],[6,1],[5,0],[5,2],[4,4]]`

| Step | p | insert at | que after insert |
|------|---------|-----------|-----------------|
| 1 | [7,0] | 0 | [[7,0]] |
| 2 | [7,1] | 1 | [[7,0],[7,1]] |
| 3 | [6,1] | 1 | [[7,0],[6,1],[7,1]] |
| 4 | [5,0] | 0 | [[5,0],[7,0],[6,1],[7,1]] |
| 5 | [5,2] | 2 | [[5,0],[7,0],[5,2],[6,1],[7,1]] |
| 6 | [4,4] | 4 | [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]] |

**Output:** `[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]` ✓

Verify `[4,4]`: people in front with height ≥ 4 are `[5,0],[7,0],[5,2],[6,1]` → exactly 4. ✓

---

## Complexity Analysis

| | Complexity |
|---|---|
| **Time** | O(n²) — each `insert` on a list is O(n) and we do it n times |
| **Space** | O(n) — the output queue |

> `list.insert` is O(n) because Python lists are backed by arrays — inserting in the middle requires shifting all elements to the right of the insertion point.

---

## Key Takeaways

1. **Greedy + sorting** is the pattern: find a sort order that makes local greedy decisions globally correct.
2. **Taller people first** means shorter people never interfere with already-placed constraints.
3. **`list.insert(i, v)`** is O(n) — fine here, but worth knowing. For very large inputs a linked list would give O(1) inserts (though Python doesn't have a built-in one).
4. The tie-breaking sort `(-h, k)` is essential: same-height people must be inserted in ascending `k` order or they'll end up in each other's way.
5. This greedy decomposition pattern — "fix one dimension, then greedily resolve the other" — also appears in problems like LC 135 Candy and LC 45 Jump Game II.
